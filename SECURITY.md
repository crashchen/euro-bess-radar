# Security Policy

## Reporting a Vulnerability

Please report suspected security issues privately to `crashchen@gmail.com`.

Do not open a public GitHub issue for:
- exploitable vulnerabilities
- credential or token exposure
- sensitive configuration mistakes
- anything that would meaningfully lower the cost of attacking a live deployment or local environment

When possible, include:
- the affected commit, branch, or release
- a short description of impact
- clear reproduction steps
- any logs, screenshots, or proof-of-concept details needed to validate the issue

## Response Expectations

This is a small project, so response times may vary. Good-faith reports will be reviewed and triaged as quickly as practical.

## Scope

This policy covers the code and configuration stored in this repository. Third-party APIs and data providers such as ENTSO-E, Elexon, Fingrid, and Regelleistung remain governed by their own platforms, terms, and security processes.
